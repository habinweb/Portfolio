GO SITE

성북문화재단 사이트 리뉴얼

Personal Project

Duration

5 Weeks

Contribution

100%

Overview

문제인식 : 기존 성북문화재단 홈페이지는 메인 페이지를 풀스크린 스크롤 섹션 중심으로 구성하고 있었으나, 레이아웃 오버플로우와 배너 이미지 비율 문제로 PC 환경에서 가독성이 떨어지는 한계가 있습니다. 또한 재단 내부의 다양한 프로그램과 독립적인 사업들이 메인 화면에서 충분히 드러나지 않아, 재단의 전체 활동이 직관적으로 인지되지 않는 구조였습니다.

분석 : 문화재단 사이트임에도 불구하고 풍부한 포스터 이미지와 시각적 자산이 효과적으로 활용되지 못하고 있었으며, 넓은 화면을 사용하는 데스크톱 환경의 장점이 충분히 반영되지 않았습니다. 콘텐츠의 양과 다양성에 비해 정보 구조가 단순해, 사용자가 재단의 여러 기관과 프로그램을 탐색하기까지 추가적인 동선이 필요하다는 점이 주요 문제로 보였습니다.

개선 방향 : 풀스크린 스크롤 중심의 구성에서 벗어나 PC 환경에 최적화된 레이아웃과 안정적인 정보 흐름을 우선으로 설계했습니다. 포스터와 이미지 콘텐츠를 넓게 배치해 갤러리와 같은 시각적 경험을 제공하고, 각 프로그램과 기관이 독립적으로 인지될 수 있도록 정보 구조를 재정비함으로써 성북문화재단의 콘텐츠 다양성과 정체성이 자연스럽게 드러나는 웹사이트를 목표로 했습니다.

What I Did & Solved

  • PC 환경에서 정보 가독성이 떨어지던 기존 구조를 파악해 데스크톱 화면에 최적화된 PC-first 레이아웃으로 전체 퍼블리싱을 진행했습니다.
  • 스타일 유지보수를 고려해 root 파일에 컬러·폰트 등 공통 스타일을 정의하고, 모든 CSS를 클래스 기반으로 관리해 추후 태그 변경 시에도 스타일이 안정적으로 유지되도록 설계했습니다.
  • 헤더·푸터·고정 네비게이션 등 반복되는 레이아웃을 분리해 js의 패치 기능을 활용하여 UI 레이아웃 컴포넌트화를 시도했습니다.
  • 페이지 수 증가로 인한 관리 이슈를 인식하고 SCSS 구조로 전환해 쪼개진 여러 스타일을 scss파일로 관리하되, 하나의 css파일로 최종 컴파일했습니다.
  • 탭 전환과 모달 등 기본 인터랙션은 직접 구현하고, 생소한 기능은 예제를 검색해 라이브러리 사용을 최소화했습니다.
  • 추후 WAVE 접근성 검사 도구를 활용해 대체 텍스트와 문서 구조 중심으로 점검·보완했습니다.

Result Screens

결과물 스크린샷 및 설명

Grid를 활용해 넓은 레이아웃 골격으로 반응형 UI 재설계

헤더/고정 퀵메뉴/메인 콘텐츠/푸터로 구성된 랜딩 페이지로, grid 및 flex를 활용해 기초 레이아웃을 설계했으며, 슬라이드와 탭 UI로 주요 정보를 한 화면에서 탐색할 수 있도록 설계했습니다. 또한 PC 환경에서의 hover 스타일이 모바일 환경과 충돌하지 않도록 @media (hover: hover)구문을 적용했습니다.

서브 페이지에서도 재사용할 수 있는 헤더 및 푸터 UI

헤더/고정 퀵메뉴/푸터 공통 레이아웃을 재사용 했으며 상단 + 아래 2컬럼(부가정보/본문) 구조로 구성한 게시글 상세 서브페이지입니다.

웹접근성을 위한 스크린리더 + 랜드마크 크로스 체크

프로젝트 마무리 후 WAVE 웹사이트를 활용해 자동 웹 접근성 검사를 진행하고, 마크업 구조와 대체 텍스트 중심으로 접근성 요소를 점검했습니다. 수동 검사가 필요한 유튜브 영역을 제외해 접근성 검사를 통과했습니다. 프로젝트가 끝난 뒤에도 윈도우 기본 스크린리더를 통한 랜드마크 크로스 체크를 진행해 웹접근성을 고려했습니다.
